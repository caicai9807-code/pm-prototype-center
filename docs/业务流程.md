# 业务流程

用于描述中后台复杂业务的端到端流程，帮助原型页面保持业务闭环。

## 建议填写规则

- 从触发条件开始，写到最终结果或归档节点。
- 明确每个节点的处理角色。
- 明确关键判断条件、分支和回退路径。
- 若同一流程存在标准流和异常流，建议分开描述。
