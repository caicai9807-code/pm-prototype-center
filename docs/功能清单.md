# PC 端功能清单

| 任务管理 | 任务详情 | 查看进度与统计 | 查看单个任务完整信息：进度步骤条（草稿→待开始→进行中→已结束，当前状态高亮）、基础信息描述、检查统计（总站点数/已完成数/合格数/不合格数）。支持导出任务和保存修改。                              | □ ✅ □ ❓ □ ✏️ |     |

## 二、任务创建与编辑

| 一级功能 | 二级功能   | 三级功能   | 功能描述                                                                                                                                             |     客户确认     | 备注  |
| :--- | :----- | :----- | :----------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | :-- |
| 任务创建 | 基础信息   | 填写任务信息 | 创建新任务，填写任务名称（必填）、所属年度（必填）、开始时间（必填）、结束时间（必填），从站点库中选择参与检查的站点（至少选1个）。支持暂存草稿。                                                                        | □ ✅ □ ❓ □ ✏️ |     |
| 任务创建 | 基础信息   | 选择站点   | 打开站点选择弹窗。**筛选条件**：支持通过【管理部】下拉选择、【片区所】下拉选择、【站点类型】下拉选择、【站点名称】模糊搜索、【是否普查】下拉选择、【是否检查】下拉选择进行筛选。支持全选和单选。已选站点以表格展示（换热站编码、换热站名称、管理部、片区所、站点类型、用热地址），支持移除。 | □ ✅ □ ❓ □ ✏️ |     |
| 任务创建 | 抽选专家   | 配置抽取规则 | 配置专家抽取规则：年龄优先条件、检查间隔、各管理部专家人数、专家库范围、全量抽取开关、抽取总数量。点击抽取后自动匹配填充专家。                                                                                  | □ ✅ □ ❓ □ ✏️ |     |
| 任务创建 | 抽选专家   | 分配专家   | 为3个管理部各分配2名检查专家，共6个槽位。专家选择弹窗支持按组织部门树筛选、按姓名搜索，单选一名专家填充到指定槽位。所有槽位填满后才能提交任务。                                                                        | □ ✅ □ ❓ □ ✏️ |     |
| 任务编辑 | 草稿/待开始 | 全量编辑   | 已创建但未开始的任务，可编辑全部内容：任务名称、年度、时间、站点列表、专家分配。                                                                                                         | □ ✅ □ ❓ □ ✏️ |     |
| 任务编辑 | 进行中    | 仅改专家   | 正在执行的任务，仅允许修改专家分配，任务名称、时间、站点列表不可修改。基础信息置灰显示。                                                                                                     | □ ✅ □ ❓ □ ✏️ |     |

## 三、专家检查（PC端）

| 一级功能 | 二级功能  | 三级功能   | 功能描述                                                                                                                                                   |     客户确认     | 备注  |
| :--- | :---- | :----- | :----------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | :-- |
| 我的任务 | 任务列表  | 查看本人任务 | 专家查看本人负责的任务列表。**列表字段**：序号、任务名称、所属年度、开始时间、结束时间、任务状态（待确认-黄/待开始-紫/进行中-蓝/已结束-灰，彩色标签）、待处理站点数（按状态高亮着色）、操作按钮。**筛选条件**：支持通过【任务名称】模糊搜索、【所属年度】下拉选择、【任务状态】下拉选择筛选。 | □ ✅ □ ❓ □ ✏️ |     |
| 我的任务 | 任务列表  | 确认任务   | 待确认状态的任务，专家点击「确认」后，状态变为待开始，等待到达开始时间后自动变为进行中。                                                                                                           | □ ✅ □ ❓ □ ✏️ |     |
| 我的任务 | 任务列表  | 拒绝任务   | 专家因故不能执行时，填写拒绝原因（必填，200字以内）后拒绝。任务变为"已拒绝/转派中"状态，触发部门领导转派流程。                                                                                             | □ ✅ □ ❓ □ ✏️ |     |
| 我的任务 | 任务列表  | 转派审核   | 部门领导查看专家拒绝后的转派任务，查看任务摘要、原专家信息、拒绝原因、转派历史日志。从本部门可选专家中选择一名新专家，确认指派后新专家默认已确认，无需再次确认。                                                                       | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 任务概览  | 查看任务信息 | 页面顶部展示当前任务整体信息：任务名称、状态、所属专家（两名）、年度、周期、下发时间、当前进度、执行状态。支持在两专家之间切换账号以演示不同视角。                                                                              | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 站点切换  | 查看站点信息 | 以标签页展示任务下所有站点，点击切换时动态刷新。站点信息展示：编码、名称、管理部、片区所、行政区、联系人、普查人员、普查日期、未入网建筑数量、当前站点状态、检查记录单状态、当前填写人、确认进度。                                                      | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 普查信息  | 查看面积数据 | 查看站点的面积对比卡片（原面积与现面积对比，含住宅/非住宅分类明细）、普查明细表格（17列：序号、建筑物名称、年代、层数、用热性质、收费类别、控制方式、暖气类型、建筑面积、地上面积、地下面积、采暖面积、非采暖面积、普查依据、附件、备注、操作）。点击「详情」弹出弹窗查看14项详细信息。         | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 检查记录单 | 填写检查项  | 7项检查清单：面积核查汇总表是否完整、用热范围平面图是否完整、面积核查依据是否有效、是否存在面积变化、汇总表与现场是否一致、商业门头平面图是否一致、实测计算是否满足制度。每项选择"是/否/不适用"，选"否"时展开子项多选。系统自动判定：任一"否"→不合格，全部"是/不适用"→合格。          | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 检查记录单 | 争抢式填写  | 两名检查人中，先点击"开始填写"的人成为当前填写人，另一人进入页面后只能只读查看，不能同时编辑。显示提示："当前检查记录单正在由XXX填写，提交确认后您可进行确认。"                                                                    | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 检查记录单 | 提交与确认  | 填写完成后提交，进入"确认中"状态，两名检查人都需要确认。填写人提交后自动标记为已同意。另一名检查人点击「同意」或「驳回」。全部同意→站点完成；任一驳回→退回填写人修改。驳回时必须填写驳回原因。                                                      | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 整改通知单 | 不合格时生成 | 检查结论为不合格时，显示整改通知单标签。填写通知单内容：编号（自动生成）、部门、站点、编码、整改事项、整改意见、整改期限（工作日，≤30日自动通过，>30日人工审核）、被检查人员、检查人员。支持编辑和导出。                                                | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 实测复核  | 实测建筑复核 | 仅普查依据为"实测"的建筑显示在此标签。支持上传实测复核照片、填写复核面积。输入面积后实时计算误差：≤500㎡允许误差±3%，500~2000㎡允许±2%，>2000㎡允许±60㎡。超限时显示红色提示，但不阻断保存。                                           | □ ✅ □ ❓ □ ✏️ |     |
| 开始检查 | 操作记录  | 追溯全流程  | 以时间线展示当前站点的所有操作：任务下发、开始填写、暂存、提交、同意、驳回、修改、重新提交、完成。驳回原因完整展示，可追溯到具体操作人、时间、说明。无记录时显示空态。                                                                    | □ ✅ □ ❓ □ ✏️ |     |

## 四、整改管理

| 一级功能  | 二级功能  | 三级功能    | 功能描述                                                                                                                                                                                                                         |     客户确认     | 备注  |
| :---- | :---- | :------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | :-- |
| 整改通知书 | 审核列表  | 查看待审核   | 业务管理员查看待审核的整改通知书列表。**列表字段**：序号、主任务名称、换热站编码、站点名称、管理部、整改期限、审核状态（待审核-黄/已通过-绿/已驳回-红）、审核人、审核时间、审核意见、操作按钮。**筛选条件**：支持通过【任务名称】模糊搜索、【站点名称】模糊搜索、【审核状态】下拉选择（全部/待审核/已通过/已驳回）筛选。                                                         | □ ✅ □ ❓ □ ✏️ |     |
| 整改通知书 | 审核操作  | 通过/驳回   | 打开审核弹窗，查看通知书详情（编号、部门、整改事项、意见、期限、状态），选择审核结果（通过/驳回），填写审核意见。通过则进入下发整改流程，驳回则退回修改。                                                                                                                                                | □ ✅ □ ❓ □ ✏️ |     |
| 整改通知书 | 审核规则  | 自动审核    | 整改期限不超过30个工作日时，系统自动审核通过；超过30个工作日时，需业务管理员人工审核。                                                                                                                                                                                | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 任务列表  | 查看本人任务  | 整改责任人查看本人负责的整改任务。**列表字段**：序号、主任务名称（仅展示，不可点击）、换热站编码、站点名称、管理部、片区所、整改人、整改状态（待整改-橙/整改中-蓝/待专家审核-紫/已完成-灰/已超期-红，彩色标签）、整改截止时间、是否超期（是/否）、超期天数、当前处理节点、操作按钮。**筛选条件**：支持通过【任务名称】模糊搜索、【站点名称】模糊搜索、【整改状态】下拉选择、【整改人】下拉选择、【管理部】下拉选择、【片区所】下拉选择、【是否超期】下拉选择（全部/是/否）筛选；管理部与片区所联动。 | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 任务列表  | 接收任务    | 待整改状态的任务，责任人点击「接收」按钮，确认接收后开始整改计时。                                                                                                                                                                                            | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 任务列表  | 提交整改    | 整改中或已超期的任务，点击「整改」按钮，填写整改情况说明（500字以内）、上传整改告知书附件（必填）。提交后状态变为待专家审核。                                                                                                                                                             | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 任务详情  | 查看信息    | 查看整改任务完整信息：基础信息（8字段）、5个标签页（普查信息/实测复核照/检查记录单/整改通知单/整改告知书）、面积对比、普查明细表格。                                                                                                                                                        | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 复查确认  | 双人确认    | 整改提交后进入待专家审核状态，两名专家各自选择同意或不同意。均同意→整改完成归档；任一不同意→退回整改中，责任人补充后重新提交。                                                                                                                                                             | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 超期处理  | 超期提示与提交 | 超过截止时间未提交的任务，状态自动变为"已超期"，红色高亮显示"超期X天"。超期后仍可提交整改，但保留超期记录。整改弹窗顶部显示红色超期提示。                                                                                                                                                      | □ ✅ □ ❓ □ ✏️ |     |
| 整改任务  | 负责人规则 | 自动分配    | 整改任务按站点所属片区所自动分配负责人：裕华片区→刘建军、长安片区→王海涛、桥西片区→赵国强。                                                                                                                                                                              | □ ✅ □ ❓ □ ✏️ |     |

# 业务流程汇总

## 主流程

```
业务管理员创建面积检查任务
  → 选择参与检查的站点
  → 抽选并分配专家（每管理部2名）
  → 提交任务（待开始状态）

专家确认任务
  → 确认 → 等待开始时间 → 进行中
  → 拒绝 → 填写原因 → 部门领导转派 → 新专家默认已确认

专家开始现场检查
  → 查看普查信息（面积对比、明细、图纸）
  → 填写检查记录单（7项检查项）
  → 系统自动判定合格/不合格
    → 合格 → 站点完成
    → 不合格 → 生成整改通知单
      → 双人确认 → 审核整改通知书
        → 通过 → 生成整改任务
          → 整改责任人接收 → 提交整改
            → 双人复查确认 → 全部同意 → 整改完成归档
```

## 异常流程

| 异常场景     | 处理方式                                        |
| :------- | :------------------------------------------ |
| 专家拒绝任务   | 填写拒绝原因 → 退回部门领导 → 指派新专家 → 新专家默认已确认          |
| 检查记录单被驳回 | 任一检查人不同意 → 退回填写人修改 → 修改后重新提交确认              |
| 整改通知书被驳回 | 任一检查人不同意 → 退回修改 → 修改后重新提交审核                 |
| 整改复查被驳回  | 任一专家不同意 → 退回整改中 → 责任人补充后重新提交                |
| 整改超期     | 超过截止时间 → 状态变为已超期 → 红色高亮显示超期天数 → 仍可提交但保留超期记录 |
