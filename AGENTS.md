# AGENTS

你是我的中后台产品原型开发助手。

## 项目定位

- 本仓库用于沉淀复杂中后台业务 HTML 原型。
- 默认采用多页面 HTML 原型包。
- 每个页面是独立 HTML 文件。
- 页面之间必须使用相对路径跳转。
- 原型用于需求评审、客户演示、研发沟通，不是生产代码。

## 设计规则

- 视觉风格参考 Ant Design。
- 页面布局紧凑，信息密度高。
- 优先模拟 `Table`、`Form`、`Tabs`、`Drawer`、`Modal`、`Descriptions`、`Tag`、`Steps` 等中后台组件。
- 字段名称和字段内容尽量不换行。
- 不得随意删减业务字段。
- 缺失字段要标记，不要自行删除。

## 目录规则

- `docs/` 存放业务说明、字段字典、流程和交互规则。
- `prototypes/` 存放可预览 HTML 原型。
- `assets/screenshots/` 存放参考截图。
- `deliverables/` 存放最终交付包。

## 省 Token 规则

- 默认只读取当前任务相关文件。
- 不要扫描全仓库。
- 检查问题时优先读取 `docs`、目标 HTML 文件和本次修改 diff。

## 完成标准

- 每次修改后说明改了哪些文件。
- 页面之间跳转必须可用。
- 输出本地打开方式。

## 本地打开方式

- 直接双击 HTML 文件打开，或在浏览器中打开目标页面文件。
- 若页面较多，优先从模块首页或导航页进入，并沿相对路径演示页面跳转。
