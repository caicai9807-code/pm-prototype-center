# 中后台复杂业务 HTML 原型仓库

本仓库用于搭建中后台复杂业务的多页面 HTML 原型包，适合产品方案表达、客户演示、需求对齐和实施交底。

## 当前阶段

当前仅初始化项目骨架和说明文档，不包含具体业务页面。

## 原型方式

- 每个业务模块一个文件夹
- 每个页面一个 HTML 文件
- 页面之间使用相对路径互相跳转
- 不使用真实后端
- 使用 mock 数据
- 风格参考 Ant Design 中后台
- 页面布局紧凑，字段完整，适合客户演示

## 目录结构

```text
.
├── AGENTS.md
├── README.md
├── assets/
│   └── screenshots/
├── deliverables/
├── docs/
│   ├── 业务流程.md
│   ├── 交互规则.md
│   ├── 字段字典.md
│   └── 页面清单.md
└── prototypes/
```

## 推荐原型组织方式

```text
prototypes/
  用户中心/
    首页.html
    列表页.html
    详情页.html
    新建页.html
    编辑页.html
    mock/
      列表数据.json
      详情数据.json
```

## 文档用途

- `docs/页面清单.md`：定义模块、页面、导航关系、页面职责
- `docs/字段字典.md`：定义字段含义、格式、来源、校验、枚举
- `docs/业务流程.md`：定义流程节点、角色分工、前后置条件
- `docs/交互规则.md`：定义联动、显隐、状态、校验、异常提示

## 后续建议

1. 先选一个核心业务模块建立首个原型目录。
2. 先补页面清单和业务流程，再开始页面制作。
3. 每完成一个模块，同步补齐字段字典和交互规则。
4. 演示前将截图沉淀到 `assets/screenshots/`，输出包沉淀到 `deliverables/`。
